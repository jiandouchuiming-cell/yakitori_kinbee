# LP公開前チェックリスト（焼き鳥 金兵衛）

モックアップ段階で残した「後で対応する」項目の抜け漏れ防止用リスト。
すべて完了するまで本番公開しないこと。

## 1. 写真の差し替え
- [ ] 店舗側との契約・撮影日程確定
- [ ] 実店舗の実メニュー・実盛り付けで写真撮影
- [ ] モックアップ内のAI生成画像（ChatGPT Image由来）をすべて実写真に差し替え
- [ ] 差し替え漏れがないか、公開直前に全セクションを目視確認
- [ ] それまでの提案・共有資料には「写真はイメージです。実際の提供メニューと異なります」の注記を入れる

## 2. 未確認情報の店舗確認
- [ ] 定休日（モックアップ仮表記「日曜・不定休」）
- [ ] 正式な営業時間・L.O.時刻（モックアップ仮表記「17:30〜23:00 (L.O.22:30)」）
- [ ] 正式なコース内容・品数
- [ ] ワインの具体的な銘柄
- [ ] 支払い方法
- [ ] キャンセルポリシー
- [ ] アレルギー・苦手食材への対応可否
- [ ] 香水・においに関する来店ルールの有無
- [ ] 上記すべて確認できるまでFAQ・店舗情報セクションは仮表記のまま公開しない

## 3. モックアップで未反映の構成要素（本実装で必ず含める）
- [ ] 2コース比較（10品5,000円／15品7,000円の違いを明記）
- [ ] 利用シーン訴求セクション（デート・記念日・会食・一人飲み）
- [ ] 予約導線に「食べログで予約」ボタンを明示（DM・電話と並べて行き先を明確化）

## 4. KPI再設計（マーケター・経営者レビュー反映）
- [ ] 既存の予約数（経路別：食べログ・Instagram DM・電話）のベースラインを店舗側にヒアリング
- [ ] KPIを「予約件数」絶対数だけでなく「LP経由の新規顧客率」「平日席占有率の変化」で測定する設計に変更（既存導線からのシフト＝カニバリを増収と誤認しないため）
- [ ] GA4等で経路別の流入・予約クリックを計測できる状態にする

## 5. DM予約の運用方針
- [ ] Instagram DMはダブルブッキング・既読放置のリスクが高いため、LPからのDM予約導線は設けず食べログ予約フォームへ一本化する
- [ ] 電話予約は営業中の一次対応者を決めておく
